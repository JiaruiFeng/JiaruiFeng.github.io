### 1. Introduction

One common thing we will think about after training an autoencoder is that, can the decoder generate good data if we just give it some random input. Like a image auto encoder, can we generate new images after we train an auto encoder using large scale dataset. The answer is not. The reason is that, during training, we don't give any constrains on the latent space, which means that the encoder and decoder can take chance to adjust their parameters to fit any input. This leads the model to severe overfitting and hard to modeling the latent distribution of data. To solve this problem, we need to put some constrains to autoencoder to make it not only can reconstruct data but also learn the distribution of data, which leads us to **Variational Autoencoder(VAE)**. VAE is a generative unsupervised model that define an intractable density function with latent variable $z$, this $z$ can help the model to avoid overfitting and ensure that model can learn a meaningful latent distribution of data. In VAE, we don't just encode data to a single point, instead, we encode the data as a distribution over the latent space. The model is trained as follow:

* First, the input is encode as distribution over the latent space.
* Second, a point from the latent space is sampled from the latent space.
* Then, the sampled point is decoded and the reconstruction error is computed
* Finally, using the error to optimize the VAE

Next, let's we see the detail of the model

### 2. Bayesian Learning

Before we dig into VAE, we first briefly discuss bayesian learning, which will be very useful when we derive the formula for VAE. Consider we have a image dataset $D$, based on dataset, we want to generate new image $x$, the probability model is:


$$
P(x|D)
$$


Using the Baye's formula, we get:


$$
P(x|D)=\frac{P(D|x)P(x)}{P(D)}
$$


Where $P(D)$ is a constant and $P(x)$ is piror knowledge. However, the real problem is that we cannot directly modeling the probability $P(D\lvert x)$. Thus, to compute $P(x\lvert D)$, we can instead assume both $D$ and $x$ is generated by series latent variable $z_i$ where $i=1,2,...,N$. Then, we can try to learn the $z$ and let $z$ to generate $x$:


$$
\begin{aligned}
P(x|D)&=\sum^N_{i=1}P(x,z_i|D)\\
&=\sum^N_{i=1}P(x|z_i,D)P(z_i|D)
\end{aligned}
$$


Then, we know that $x$ and $D$ are independent given $z_i$, using Conditional independence, we have:


$$
\begin{aligned}
P(x|D)&=\sum^N_{i=1}P(x|z_i,D)P(z_i|D)\\
&=\sum^N_{i=1}\frac{P(x,D|z_i)}{P(D|z_i)}P(z_i|D)\\
&=\sum^N_{i=1}\frac{P(x|z_i)P(D|z_i)}{P(D|z_i)}P(z_i|D)\\
&=\sum^N_{i=1}P(x|z_i)P(z_i|D)
\end{aligned}
$$


$P(x\vert z_i)$ can be computed very easily once we learn the distribution of $z_i$, $P(z_i\vert D)$  can be computed by:


$$
P(z_i|D)=\frac{P(D|z_i)P(z_i)}{P(D)}=P(D|z_i)P(z_i)
$$


During learning, We want to learn the $z$ based on the $D$:


$$
max \ P(D)=\sum^N_{i=1}P(D,z_i)
$$


We can see that this is exact the form that can be solving using EM algorithm of we know the $N$ and distribution type of distribution of $z_i$. To see this, let the parameter of distribution $z$ be $\theta$:


$$
max\ P(D)=P(D|\theta)= \sum^N_{i=1}P(D,z_i|\theta)
$$



### 3.Method of VAE

In VAE, what we want to learn is maximum the probability of data $x$ based on latent variable $z$, which is exactly the formlua of Bayesian Learning. Let $\theta$ be parameter of decoder and $P_{\theta}(x|z)$ be the probability of decoder to generate the data $x$:


$$
max \ p_{\theta}(x)=\int p_{\theta}(x|z)p_{\theta}(z)dz=\mathbf{E}_{z \sim p_{\theta(z)}}p_{\theta}(x|z)
$$


However, here we cannot compute it directly since the distribution  is intractable. One idea is to using posterior distribution to approximate $p_{\theta}(z)$. Let the  $p_{\theta}(z|x)$ be the posterior distribution modeling by decoder, which equal to:


$$
p_{\theta}(z|x)=\frac{p_{\theta}(x|z)p_{\theta}(z)}{p_{\theta}(x)}
$$


If we know $p_{\theta}(z\lvert x)$, the $p_{\theta}(x)$ can be computed by:


$$
p_{\theta}(x)=\int p_{\theta}(z|x)p(x)dz= \mathbf{E}_{z \sim p_{\theta(z|x)}}p_{\theta}(x)
$$


However, the $p_{\theta}(z \lvert x)$ is still intractable. To solve this problem, we notice we not only have decoder, but also encoder. let the parameter in encoder be $\phi$. the distribution in encoder can be $q_{\phi}(z\lvert x)$, What if we apporximate $p_{\theta}(z\lvert x)$ using $q_{\phi}(z \lvert x)$. To make it easy, we use log function:


$$
\begin{aligned}
log\ p_{\theta}(x)&=\mathbf{E}_{z \sim q_{\phi(z|x)}}log\ p_{\theta}(x)\\
&=\mathbf{E}_{z \sim q_{\phi(z|x)}}log\ \frac{p_{\theta}(x|z)p_{\theta}(z)}{p_{\theta}(z|x)}\\
&=\mathbf{E}_{z \sim q_{\phi(z|x)}}log\ \frac{p_{\theta}(x|z)p_{\theta}(z)}{{p_{\theta}(z|x)}}\frac{q_{\phi}(z|x)}{q_{\phi}(z|x)}\\
&=\mathbf{E}_{z \sim q_{\phi(z|x)}}log\ p_{\theta}(x|z)-\mathbf{E}_{z \sim q_{\phi(z|x)}}log\ \frac{q_{\phi}(z|x)}{p_{\theta}(z)}+\mathbf{E}_{z \sim q_{\phi(z|x)}}log\ \frac{q_{\phi}(z|x)}{p_{\theta}(z|x)}\\
&=\mathbf{E}_{z \sim q_{\phi(z|x)}}log p_{\theta}(x|z)-D_{KL}(q_{\phi}(z|x)||p_{\theta}(z))+D_{KL}(q_{\phi}(z|x)||p_{\theta}(z|x))
\end{aligned}
$$


The first term is just reconstruction loss, which can be estimated by sampling(talk about later). The second KL term measure the difference between prior distribution $p_{\theta}(z)$ and posterior distribution $q_{\phi}(z \lvert x)$, which also have nice closed-form solution.  The problem is the third term, where we need to to compute $p_{\theta}(z \lvert x)$. But we notice that we can actually optimze the formula without computing this term, let the lower bound $l_b$ be:


$$
l_b=\mathbf{E}_{z \sim q_{\phi(z|x)}}log p_{\theta}(x|z)-D_{KL}(q_{\phi}(z|x)||p_{\theta}(z))
$$


The probability $log\ p_{\theta}(x)$ can view as $l_b$ add an additional term. If we first fix all the thing but just optimize $q_{\phi}(z \lvert x)$, since $p_{\theta}(x)$ is indenpent to $q_{\phi}(z \lvert x)$, the change of  $q_{\phi}(z \lvert x)$ doesn't change the value of $log\ p_{\theta}(x)$, but only change the propotion of $l_b$ and $D_{KL}$. Thus we can apporximate the $log\ p_{\theta}(x)$ using $l_b$ by changing $q_{\phi}(z \lvert x)$ to approximate $p_{\theta}(z \lvert x)$, then we can maximize th $l_b$ by changing the $p_{\theta}(x\lvert z)$.

<img src="/assets/images/image-20200625144439072.png" alt="image-20200625144439072" style="zoom:50%;" />


$$
log\ p_{\theta}(x)=l_b+D_{KL}(q_{\phi}(z|x)||p_{\theta}(z|x))\geq l_b=\mathbf{E}_{z \sim q_{\phi(z|x)}}log p_{\theta}(x|z)-D_{KL}(q_{\phi}(z|x)||p_{\theta}(z))
$$


This is called variational inference.

### 4. Architecture of VAE

After we have the theory base of VAE, we now talk about how to implement it using autoencoder. Actually, the architecture of VAE is pretty similar to normal autoencoder:

<img src="/assets/images/image-20200625180840146.png" alt="image-20200625180840146" style="zoom:50%;" />

The difference is lying on the output of encoder and input of decoder. In order the encoder to modeling the distribution $q_{\phi}(z\lvert x)$, the output of encoder is no longer latent value but mean $\mu_{z|x}$ and variance $\Sigma_{z|x}$of latent distribution $q_{\phi}(z\lvert x)$.  However, if we directly input mean and variance to the decoder, decoder cannot learn the distribution and the VAE is actually degenerated to normal autoencoder. Instead, we need to sample a new data from the distribution $q_{\phi}(z \lvert x)$. Let $\epsilon \sim \mathcal{N}(0, I)$, the sampling procedure is:


$$
z=\mu_{z \mid x}+\epsilon \Sigma_{z \mid x}
$$


Then, we input $z$ into decoder and let the decoder learn the distribution $p_{\theta}(x \lvert z)$ by reconstruct original data $x$. However, it still cannot learn the distribution is we just train this architecture, since we don't have any constrain on the distribution $q_{\phi}(z \lvert x)$. Anything will tend to be simple naturally if we don't put control on them. If you don't constrain $q_{\phi}(z \lvert x)$, during training, the $\Sigma_{z|x}$ will become zero and the VAE degenerate to normal autoencoder.  Remember that in $l_b$, we need to minimize term $D_{KL}(q_{\phi}(z\lvert x)\lvert \rvert p_{\theta}(z))$, which can be achieve when $q_{\phi}(z \lvert x)$ approximate to $p_{\theta}(z)$. Thus, to constrian the distribution, we can let the prior distribution $p_{\theta}(z)$ be the centered isotropic multivariate Gaussian $p_{\theta}(z)=\mathcal{N}(\mathbf{z}: \mathbf{0}, \mathbf{I})$. This time, druing training, the $q_{\phi}(z\lvert x)$ will try to approximate $p_{\theta}(z)$ and become Gaussian distribution.

Finally, we talk about how to compute $l_b$ in VAE. recall that the loss is:


$$
\begin{aligned}
min\ -l_b&=-\mathbf{E}_{z \sim q_{\phi(z|x)}}log p_{\theta}(x|z)+D_{KL}(q_{\phi}(z|x)||p_{\theta}(z))\\
&=-\mathbf{E}_{z \sim q_{\phi(z|x)}}log p_{\theta}(x|z)+D_{KL}(q_{\phi}(z|x)||\mathcal{N}(z: 0, I))
\end{aligned}
$$


The first term is just reconstruction loss, which can be optimized by minimize cross entropy. The compute second term, we can see that:


$$
\begin{aligned}
D_{KL}(q_{\phi}(z|x)||\mathcal{N}(z: 0, I))&=\int q_{\phi}(z|x)log\ \frac{q_{\phi}(z|x)}{\mathcal{N}(z: 0, I))}dz\\
\end{aligned}
$$


If we also assume that $q_{\phi}(z \lvert x)$ is gaussian distribution with mean $\mu$ and variance $\sigma^2$, we can have nice solution of above:


$$
\begin{aligned}
D_{KL}(q_{\phi}(z|x)||\mathcal{N}(z: 0, I))
&=\int q_{\phi}(z|x)log\ \frac{q_{\phi}(z|x)}{\mathcal{N}(z: 0, I))}dz\\
&=\int \frac{1}{\sqrt{2 \pi \sigma^{2}}} e^{-(z-\mu)^{2} / 2 \sigma^{2}}\left(\log \frac{e^{-(z-\mu)^{2} / 2 \sigma^{2}} / \sqrt{2 \pi \sigma^{2}}}{e^{-z^{2} / 2 / \sqrt{2 \pi}}}\right) d z\\
&=\frac{1}{2} \int \frac{1}{\sqrt{2 \pi \sigma^{2}}} e^{-(z-\mu)^{2} / 2 \sigma^{2}}\left[-\log \sigma^{2}+z^{2}-(z-\mu)^{2} / \sigma^{2}\right] d z \\
&=\frac{1}{2}(-\log\sigma^2+\sigma^2+\mu^2-1)
\end{aligned}
$$
 

Now, we have everything for constructing a VAE. Notice that, when implement the VAE, to avoid $\sigma^2_{z|x}$ to be negative, what the ouput of encoder is actually $log\ \sigma^2_{z|x}$. The sampling function become:


$$
z=\mu_{z \mid x}+\epsilon\exp(\log\sigma^2_{z \mid x})
$$
