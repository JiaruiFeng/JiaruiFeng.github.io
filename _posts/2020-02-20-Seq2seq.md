### 1. Introduction

Sequence to sequence model is a framework that use RNN to construct, which can help to map an input sequence to an output sequence, the length for two sequence may not equal to each other. This technique allow us to use nerual network to do machine translation.

At a high level, a sequence-to-sequence model is an end-to-end model made up of two recurrent neural networks:

**Encoder**:  take an input sequence and generate a fixed-size context vector to represent input sequence.

**Decoder**: Use the context vector get in encoder to generate output sequence.



### 2. Architecture

##### Encoder

Encoder RNN produces an context vector $C$ of the source sentence using RNN structure. Here, we use LSTM to illustrate it:

<img src="/assets/images/image-20200317165612219.png" alt="image-20200317165612219" style="zoom:50%;" />

We input the sectence to the encoder(usually multi-layer RNN structure), then, the final hidden state of encoder would be the context vector that encoder all the information of this sectence. Notice that here we reverse the order of input sectence in order to get better performance in translation. Due to the gradient vanishing and long-term dependence, it is hard for context vector to encoder information in the beginning, which would compromise the performance of decoder in the first word. Usually, the first word would also be the most important word since all the words come following are depending on it. If we reverse the order of sectence, we can connect the first word of input with the first word of output directly.

##### Decoder

Decoder RNN is a Language Model that generates target sentence, conditioning on the context vector generated by encoder. Here is the structure:

<img src="/assets/images/image-20200317171418183.png" alt="image-20200317171418183" style="zoom:50%;" />

Here "GO" is a special token to signify the start of output generation; in literature, this is usually an token appended to the end of the input (there’s also one at the end of the output). In each time step $t$, the multi-layer LSTM will generate a $y_t$, which is a one-hot vector indicate the probability of each word in corpus. After we choose a word based on $y_t$, we will use the word embedding as the input of next time step $t+1$, which is also illustrate in the picture.

As we said, the decoder is also a conditional language model. To see that, let $y_t$ be the output word for time step $t$ and $x_q$ be the input words in time step $q$, where $t=1,2,...,T$ and $q=1,2,...,Q$. The probability model of seq2seq is:


$$
p\left(\mathrm{y}_{1}, \ldots, \mathrm{y}_{T} | \mathrm{x}_{1}, \ldots, \mathrm{x}_{Q}\right)
$$


Let context vector be $C$, this formula can be written as:


$$
p\left(\mathrm{y}_{1}, \ldots, \mathrm{y}_{T} | \mathrm{x}_{1}, \ldots, \mathrm{x}_{Q}\right)=\prod_{t=1}^{T} p\left(\mathrm{y}_{t} | \mathrm{C}, \mathrm{y}_{1}, \ldots, \mathrm{y}_{t-1}\right)
$$


$$ p(\mathrm{y}_{t} \lvert \mathrm{C}, \mathrm{y}_{1}, \ldots, \mathrm{y}_{t-1})$$ is the probability of output word in time step $t$ given the output words so far the input sectence, which is encoded as $C$.

##### Training

Similar to the traditional RNN, we can define the loss function for seq2seq as $J$, where:


$$
J=\frac{1}{T} \sum_{t=1}^{T} J_{t}
$$


where $J_t$ is the cross-entropy loss in decoder time step $t$.

The training of seq2seq is **end-to-end**.



### 3. decoding methods

As we mention above, in each time step of decoder, we will generate a one-hot vector indicate the probability for every word in corpus for this step. How to choose the word based on the one-hot vector? An intutitive way is to select the word with the maximum probability. This is called **Greedy Decoding**. However, there are some problems for this method. the word with the maximum probability in this step may not be the optimal word for the whole sectence. To solve this, Instead, we can compute all the possible word in each step and select the sectence with the maximum overall probability, which is called **Exhaustive Search Decoding**. However, it is obvious that this is an computation expansive way, the complexity of this method is $O(V^T)$, where $V$ is the number of words in corpus and $T$ is the number of time steps.  Now, we introduce a new method which can solve the problem occured in both.

**Bean Search Decoding**

The main idea behind bean search is that: in each time step, we keep tracking the $k$ most probable partial translations which we call it hypothesis. $k$ is called **bean size** , usually, we would like to choose 5 to 10.

For each hypothesis $h$, we have a score calculated by log probability:


$$
\operatorname{score}\left(h\right)=\log P_{\mathrm{LM}}\left(y_{1}, \ldots, y_{t} | x\right)=\sum_{i=1}^{t} \log P_{\mathrm{LM}}\left(y_{i} | y_{1}, \ldots, y_{i-1}, x\right)
$$


Here is an example using $k=2$:

![image-20200317175303343](/assets/images/image-20200317175303343.png)

In each time step, we compute the score for hypothesis and only keep 2 hypothesis with highest score. Finally, we will choose the sectence with the highest score, indicated by green. It is obvious that bean search is not guarantee to find the optimal solution, but it is much efficient than exhaustive search.

Another problem related to bean search is that here we have multiple sectence, usually it may not have same length. since a long sectence would always have low score, how to choose the final output, a simple way to solve is to normalize it based on length:


$$
score_{normal}=\frac{1}{t} \sum_{i=1}^{t} \log P_{\mathrm{LM}}\left(y_{i} | y_{1}, \ldots, y_{i-1}, x\right)
$$
